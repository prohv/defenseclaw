Connectors are the adapter layer between agent frameworks and DefenseClaw's guardrail proxy. Each connector owns all security surfaces for its agent: LLM traffic routing, tool call inspection, agent hook events, component scanning, CodeGuard file scanning, and subprocess enforcement.

## The two families

<Cards>
  <Card title="Proxy connectors" description="OpenClaw, ZeptoClaw. DefenseClaw sits in the LLM data path — every request and response is intercepted, scored, and forwarded." />
  <Card title="Hook connectors" description="Claude Code, Codex, Cursor, Windsurf, Gemini CLI, GitHub Copilot CLI, OpenHands, Antigravity, Hermes. DefenseClaw wires into the agent's native lifecycle hooks; the agent talks directly to its upstream." />
  <Card title="Compatibility contracts" href="/docs/connectors/compatibility" description="Versioned hook contracts, setup-time connector version checks, and the runtime hook_contract_lock.json." />
</Cards>

<Callout type="info">
**One gateway, many hook connectors.** A single DefenseClaw gateway can serve several hook connectors at once, each with its own guardrail posture under `guardrail.connectors.<name>` — pick **Add** (not Replace) when you run a second `setup <connector>`. Proxy connectors (OpenClaw, ZeptoClaw) bind a listener and own the traffic plane, so they can't be multi peers. See [Multi-connector](/docs/setup/guardrail/multi-connector).
</Callout>

## How a connector is structured

<Flow
  direction="TB"
  caption="Every connector implements the same Go interface; the gateway routes traffic generically once a connector has resolved its signals."
>
  <Node id="setup" kind="connector">{`Setup(ctx, opts)\nwrite hook scripts,\nback up agent files,\nregister routes`}</Node>
  <Node id="auth" kind="connector">{`Authenticate(r)\nverify gateway token`}</Node>
  <Node id="route" kind="connector">{`Route(r, body)\nextract upstream + model\n+ stream + headers`}</Node>
  <Node id="teardown" kind="connector">{`Teardown(ctx, opts)\nrestore agent files`}</Node>
  <Node id="verify" kind="connector">{`VerifyClean(opts)\nno residual artifacts`}</Node>
  <Edge from="setup" to="auth" />
  <Edge from="auth" to="route" />
  <Edge from="route" to="teardown" />
  <Edge from="teardown" to="verify" />
</Flow>

The interface is defined in [`internal/gateway/connector/connector.go`](https://github.com/cisco-ai-defense/defenseclaw/blob/main/internal/gateway/connector/connector.go); each per-connector file (`claudecode.go`, `codex.go`, `cursor.go` via `hook_only.go`, ...) implements it.
